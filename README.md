# ToyShare 🧸

> Share joy, one toy at a time.

ToyShare is a Flutter app that lets neighbors share, lend, and donate toys to children in their community.

---

## Features

- **Auth** — Sign up, login, password reset
- **Browse Toys** — Search, filter by type, tap owner to see public profile & ratings
- **List a Toy** — Add photo, condition, category, age group, optional fee
- **Exchange Duration** — Set by date+time or quick duration (5 min, 1 hr, 2 hrs, etc.)
- **QR Receipt** — Every exchange generates a QR code receipt with toy name, owner, return date
- **QR Scanner** — Scan another user's QR to verify the exchange
- **Incident Report** — Report lost, stolen, or damaged toy directly from the receipt screen
- **Chat** — In-app messaging per exchange with unread badges
- **Smart Notifications** — Return reminders at 3 days, 1 day, same day, overdue, 30-min warning for hourly loans
- **Ratings & Reviews** — Leave feedback after exchange; public profiles show ratings
- **Dark Mode** — Full dark mode across every screen
- **Web Support** — Runs on web with responsive mobile layout

## Setup

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/toyshare.git
cd toyshare
flutter pub get
```

### 2. Firebase Setup

**Install FlutterFire CLI:**
```bash
npm install -g firebase-tools
dart pub global activate flutterfire_cli
```

**Connect to your Firebase project:**
```bash
flutterfire configure
```
This generates `lib/firebase_options.dart` automatically.

**Also copy these from your Firebase project:**
- `google-services.json` → place at `android/app/google-services.json`
- `GoogleService-Info.plist` → place at `ios/Runner/GoogleService-Info.plist` (iOS only)

## Firebase Collections (Firestore)

These collections are created automatically when the app runs — no manual creation needed. Here is what each one stores:

| Collection | Purpose | Key Fields |
|---|---|---|
| `toys` | All toy listings | `ownerId`, `ownerName`, `name`, `description`, `condition`, `category`, `ageGroup`, `type` (exchange/donation), `isAvailable`, `receiveDate`, `returnDate`, `fee`, `imageUrl`, `createdAt` |
| `requests` | Borrow requests sent by users | `toyId`, `toyName`, `ownerId`, `requesterId`, `requesterName`, `status` (pending/accepted/rejected), `message`, `createdAt` |
| `exchanges` | Active and completed exchanges | `toyId`, `toyName`, `ownerId`, `ownerName`, `holderId`, `holderName`, `type`, `status` (active/pending_receipt/completed/disputed), `receiveDate`, `returnDate`, `durationMode`, `fee`, `startedAt`, `completedAt`, `disputeId`, `disputeReason` |
| `messages` | Chat messages per exchange | `exchangeId`, `senderId`, `senderName`, `receiverId`, `text`, `isRead`, `createdAt` |
| `notifications` | In-app notifications | `userId`, `title`, `body`, `type`, `relatedId`, `isRead`, `createdAt` |
| `feedback` | Reviews left after exchanges | `fromUserId`, `fromUserName`, `toUserId`, `exchangeId`, `toyName`, `isPositive`, `rating`, `comment`, `createdAt` |
| `disputes` | Reported exchange disputes | `exchangeId`, `reporterId`, `reporterName`, `otherPartyId`, `otherPartyName`, `reason`, `status` (open/resolved), `createdAt`, `resolvedAt` |
| `incidents` | Lost/stolen/damaged reports | `exchangeId`, `toyName`, `reporterId`, `reporterName`, `ownerName`, `holderName`, `incidentType` (lost/stolen/damaged), `description`, `status`, `createdAt` |
| `blacklist` | Blocked users | `userId`, `blockedUserId`, `blockedUsername`, `createdAt` |

## Firestore Rules

Firebase Console → **Firestore** → **Rules** tab → paste and publish:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isAuth() { return request.auth != null; }

    match /toys/{id}          { allow read, write: if isAuth(); }
    match /requests/{id}      { allow read, write: if isAuth(); }
    match /exchanges/{id}     { allow read, write: if isAuth(); }
    match /messages/{id}      { allow read, write: if isAuth(); }
    match /notifications/{id} { allow read, write: if isAuth(); }
    match /feedback/{id}      { allow read, write: if isAuth(); }
    match /blacklist/{id}     { allow read, write: if isAuth(); }
    match /disputes/{id}      { allow read, write: if isAuth(); }
    match /incidents/{id}     { allow read, write: if isAuth(); }
  }
}
```

## Firebase Storage Rules

Firebase Console → **Storage** → **Rules** tab → paste and publish:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /toy_images/{userId}/{imageId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
      allow delete: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

> **Note:** Storage is only needed if you enable actual photo uploads. Currently photos are stored in memory only.
